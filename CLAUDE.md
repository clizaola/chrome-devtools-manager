# chrome-devtools-manager

## Project Overview
A bash CLI tool that manages Chrome DevTools MCP (https://github.com/ChromeDevTools/chrome-devtools-mcp) across multiple projects. Single command for launching Chrome, managing ports, detecting conflicts, and configuring projects.

## Installation
- Script lives at `~/Code/chrome-devtools-manager/chrome-devtools-manager.sh`
- Symlinked to `~/.local/bin/chrome-devtools-manager` for PATH access
- Config/registry file: `~/.chrome-devtools-manager`

## Commands
| Command | Description |
|---------|-------------|
| `chrome-devtools-manager` | Launch Chrome for current project (reads port from `.mcp.json`) |
| `chrome-devtools-manager --list` | Show port registry table |
| `chrome-devtools-manager --scan` | Rescan all projects, rebuild registry, detect conflicts |
| `chrome-devtools-manager --add [port]` | Add chrome-devtools to current project's `.mcp.json` (requires `jq`) |
| `chrome-devtools-manager --remove` | Remove chrome-devtools from current project's `.mcp.json` (requires `jq`) |
| `chrome-devtools-manager --doctor` | Non-destructive health check: reports STALE/DRIFT/WARN/CONFLICT/WRONG-PORT/DEAD/ORPHAN, no file or registry mutations |
| `chrome-devtools-manager --clean [--all] [--force]` | Delete `/tmp/chrome-<project>` profile dir(s). Defaults to **current project only**; `--all` cleans every profile; `--force` kills any active Chrome before deleting |
| `chrome-devtools-manager --path <dir>` | Add a search path for project discovery |
| `chrome-devtools-manager --help` | Show help |

## How It Works
1. **Port detection:** Scans `.mcp.json` files for `127.0.0.1:<port>` or `localhost:<port>` pattern via grep
2. **Chrome isolation:** Each project gets `--remote-debugging-port` and `--user-data-dir=/tmp/chrome-<project>`
3. **Profile-singleton guard:** Before launching, inspects running Chrome processes via `ps` and refuses to launch if another Chrome is already holding the same user-data-dir on a different port (Chrome silently ignores `--remote-debugging-port` on a second invocation against the same profile — this was a real foot-gun until fixed)
4. **Launch verification:** After launching, polls `http://127.0.0.1:<port>/json/version` via `curl` (up to ~5s) to confirm the CDP endpoint is actually live before returning success
3. **OS detection:** Auto-detects macOS, Linux, WSL2 and finds the correct Chrome executable
4. **Config file (`~/.chrome-devtools-manager`):** Two sections — search paths (user-editable) and registry (auto-generated by --scan)
5. **Port auto-assignment:** `--add` without a port picks next available (starting from 9222)

## Platform Support
- **macOS:** Fully tested, works out of the box
- **Linux:** Auto-detects google-chrome, google-chrome-stable, chromium-browser, chromium
- **WSL2:** Detects via `/proc/version`, uses Windows Chrome at `/mnt/c/Program Files/Google/Chrome/Application/chrome.exe` (untested)
- **Windows native:** Not supported (requires bash shell)

## Dependencies
- `bash` — core script
- `jq` — required for `--add` and `--remove` (JSON manipulation)
- `curl` — optional, enables post-launch CDP verification and `[DEAD]` liveness checks in `--doctor`; script still works without it (verification is silently skipped)
- `ps`, `grep`, `sed`, `sort`, `uniq` — standard Unix tools

## Key Design Decisions
- **grep-based port detection** (not JSON parsing) — works regardless of JSON formatting, no `jq` dependency for basic operations
- **Config file is both config AND registry** — search paths are user-editable, registry is auto-generated
- **`--add` auto-scans after adding** — registry is always up to date
- **Temp profiles in `/tmp/`** — cleared on reboot (intentional: keeps things clean)
- **Conflict detection** — warns when two projects share a port

## Caveats
1. Uses `127.0.0.1` / `localhost` pattern matching — other hostnames (`0.0.0.0`, custom domains) won't be detected
2. Only reads first matching host:port pair per `.mcp.json`
3. Search paths must specify depth explicitly (`~/Code/*` vs `~/Code/*/*`)
4. macOS Chrome path is hardcoded to default location
5. `/tmp/` profiles lost on reboot
